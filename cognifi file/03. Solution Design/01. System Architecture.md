> User bertanya tentang keputusan investasi.
> Sistem mendeteksi bias yang sedang aktif,
> menarik data historis nyata,
> dan mengembalikan output berbasis fakta
> bukan opini, bukan rekomendasi, hanya data.

## Prinsip Desain Sistem

Sebelum arsitektur, tiga prinsip yang tidak boleh dilanggar:

**1. Data beats opinion**
Sistem tidak pernah bilang "jangan beli" atau "beli sekarang."
Sistem hanya menunjukkan apa yang terjadi di masa lalu
pada kondisi yang serupa.

**2. Intervensi bukan blokir**
Sistem tidak memblokir keputusan user.
Sistem menciptakan jeda, cukup untuk System 2 aktif.

**3. Transparent by design**
Setiap output harus bisa di-trace:
dari mana datanya, berapa episodenya, apa metodologinya.
Tidak ada black box.

## Komponen Detail

### 1. Input Layer
**Apa yang diterima:**
Natural language dari user, tidak perlu format khusus.

**Yang diekstrak:**
- Ticker saham yang disebut (GOTO, BBCA, TLKM, dll)
- Sinyal urgensi ("kayaknya mau naik", "udah masuk belum")
- Konteks posisi ("udah rugi tapi nunggu balik modal")
- Pola query (pertama kali tanya atau sudah berulang)

**Tech:**
Python + regex + LLM untuk entity extraction.

### 2. Bias Detection Engine
**Apa yang dilakukan:**
Mengklasifikasikan bias mana yang sedang aktif
berdasarkan input user dan kondisi harga saat ini.

**Tiga bias yang dideteksi:**

| Bias              | Sinyal Utama                    | Data Pendukung             |
| ----------------- | ------------------------------- | -------------------------- |
| FOMO              | Urgensi, harga naik signifikan  | % kenaikan N hari terakhir |
| Loss Aversion     | Posisi rugi, averaging down     | Durasi hold, % drawdown    |
| Confirmation Bias | Query searah, tidak ada counter | Pola pertanyaan user       |

**Output:**
```python
{
  "bias_detected": "FOMO",
  "confidence": 0.87,
  "signals": [
    "harga naik 31% dalam 5 hari",
    "kata kunci urgensi terdeteksi",
    "tidak ada pertanyaan tentang risiko"
  ]
}
```

### 3. Counter Evidence Engine
**Apa yang dilakukan:**
Menarik data historis nyata dan menghitung
probabilitas berbasis episode serupa di masa lalu.

**Sumber data:**
Yahoo Finance via `yfinance` gratis, reliable,
covers semua saham IDX liquid dengan suffix `.JK`

**Logika kalkulasi:**
```python
# Untuk FOMO detection:
# 1. Ambil semua episode: harga naik > threshold dalam N hari
# 2. Hitung outcome 30 hari setelah setiap episode
# 3. Hitung: berapa yang koreksi? seberapa dalam?
# 4. Output: probabilitas + rata-rata koreksi

episodes = find_episodes(ticker, threshold=0.25, window=5)
outcomes = calculate_outcomes(episodes, forward_days=30)
probability = len([o for o in outcomes if o < 0]) / len(outcomes)
avg_correction = mean([o for o in outcomes if o < 0])
```

**Output:**
```python
{
  "ticker": "GOTO.JK",
  "episodes_found": 6,
  "corrections": 5,
  "probability": 0.83,
  "avg_correction": -0.28,
  "data_range": "2022-2026"
}
```

### 4. Intervention Layer
**Apa yang dilakukan:**
Menggabungkan bias detection + counter evidence
menjadi output yang clear, berbasis data,
dan tidak menggurui.

**Format output:**
```
⚠️ [NAMA BIAS TERDETEKSI]

[KONDISI SAAT INI]
[Ticker] naik/turun X% dalam N hari terakhir.

[COUNTER EVIDENCE]
Data historis [Ticker].JK ([tahun]–2026):
→ X kejadian serupa
→ Y dari X diikuti koreksi dalam 30 hari
→ Rata-rata koreksi: Z%
→ Probabilitas historis: P%

[PERTANYAAN REFLEKTIF]
[Satu pertanyaan — bukan ceramah]
```

**Yang tidak ada di output:**
- Rekomendasi beli atau jual
- Prediksi harga
- Opini tentang aset

### 5. UI Layer (Streamlit)
**Komponen:**
- Input box: natural language
- Output panel: bias + counter evidence
- Chart: visualisasi episode historis
- Sidebar: pilih ticker + parameter

**Deploy:**
Streamlit Community Cloud, URL publik, gratis.

## Batasan Sistem

**Yang sistem ini BISA lakukan:**
- Deteksi pola FOMO, loss aversion, confirmation bias
- Sajikan counter-evidence berbasis data historis
- Ciptakan friction sebelum keputusan dieksekusi
- Bekerja untuk saham IDX yang terdaftar di Yahoo Finance

**Yang sistem ini TIDAK BISA lakukan:**
- Prediksi harga masa depan
- Integrasi real-time dengan broker
- Jaminan akurasi untuk saham dengan data historis pendek
- Menggantikan financial advisor profesional

**Disclaimer yang selalu muncul di output:**
> Sistem ini bukan rekomendasi investasi.
> Output berbasis data historis
> masa lalu tidak menjamin masa depan.
> Keputusan investasi sepenuhnya tanggung jawab user.

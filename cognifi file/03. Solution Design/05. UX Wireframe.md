## Tujuan File Ini

Menggambarkan flow pengalaman user dari awal hingga akhir
bagaimana sistem terlihat dan terasa saat digunakan.

File ini adalah blueprint langsung untuk build Streamlit UI.

## Prinsip UX

### Sesederhana mungkin
User target adalah Reza seorang investor muda
yang tidak punya waktu dan kesabaran untuk
mempelajari interface yang kompleks.

### Mobile-first thinking
Reza kemungkinan besar mengakses dari HP
bukan laptop. Streamlit sudah responsive,
tapi layout harus didesain dengan layar kecil
sebagai prioritas.

### Output lebih penting dari input
80% perhatian desain ada di output
bagaimana data disajikan, seberapa mudah dibaca,
seberapa jelas friction mechanism-nya.

**Implementasi Streamlit:**
```python
# Color coding per bias
bias_colors = {
    "FOMO": "error",           # merah
    "LOSS_AVERSION": "warning", # kuning  
    "CONFIRMATION_BIAS": "info" # biru
}

bias_icons = {
    "FOMO": "⚠️",
    "LOSS_AVERSION": "📉",
    "CONFIRMATION_BIAS": "🔍"
}

bias = bias_result['bias_detected']

if bias:
    container = getattr(st, bias_colors[bias])
    container(
        f"{bias_icons[bias]} **{bias} "
        f"Pattern Terdeteksi**  \n"
        f"Confidence: "
        f"{bias_result['confidence']:.0%}"
    )
    
    st.markdown("**Sinyal yang ditemukan:**")
    for signal in bias_result['signals']:
        st.markdown(f"• {signal}")

else:
    st.success(
        "Tidak ada pola bias signifikan "
        "yang terdeteksi dari input ini."
    )
```

# Processing + Output
if analyze and user_input:
    with st.spinner("Menganalisis..."):
        bias_result = detect_bias(user_input, ticker)
        engine = CounterEvidenceEngine(ticker)

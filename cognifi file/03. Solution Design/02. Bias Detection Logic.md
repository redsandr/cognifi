## Tujuan File Ini

Menjelaskan secara presisi bagaimana sistem mengidentifikasi
bias kognitif yang sedang aktif pada investor
berdasarkan sinyal yang bisa diobservasi dan diukur.

## Prinsip Deteksi

### Sistem tidak membaca pikiran
Sistem tidak tahu *apa yang user pikirkan*.
Sistem hanya mengobservasi *apa yang bisa diukur*:
- Kata-kata yang user tulis
- Kondisi harga aset saat ini
- Pola historis harga aset

Dari tiga sumber ini, sistem menghitung probabilitas
bias mana yang paling mungkin sedang aktif.

### Deteksi berbasis sinyal,
User tidak akan pernah bilang "saya sedang FOMO."
Sistem harus mendeteksi dari sinyal tidak langsung
sama seperti dokter mendiagnosis dari gejala,
bukan dari pengakuan pasien.
### Output adalah probabilitas, bukan kepastian
Sistem tidak mengklaim "kamu pasti FOMO."
Sistem mengklaim "pola input ini memiliki
87% kemiripan dengan pola FOMO historis."

## Arsitektur Deteksi

```python
def detect_bias(user_input: str, ticker: str) -> dict:

        "primary_bias": primary_bias,
        "confidence": bias_scores[primary_bias],
        "signals": get_active_signals(text_signals,

    }
```

## Deteksi Per Bias

### BIAS 1: FOMO Detection

#### Definisi Operasional
FOMO terdeteksi ketika kombinasi sinyal text
dan sinyal harga menunjukkan bahwa user
bereaksi terhadap momentum harga
bukan terhadap perubahan fundamental aset.

#### Sinyal Text (Layer 1)
```python
FOMO_TEXT_SIGNALS = {
    
    # Kata kunci urgensi
    "urgency_keywords": [
        "mau naik", "bakal pump", "sekarang", 
        "buruan", "udah telat belum", "masih sempet",
        "kayaknya mau", "feeling", "feeling kuat",
        "semua orang", "pada beli", "rame banget"
    ],
    
    # Kata kunci social proof
    "social_proof_keywords": [
        "temen gua", "kata influencer", "di grup",
        "pada profit", "semua hijau", "viral",
        "trending", "rame di tiktok", "kata si X"
    ],
    
    # Absensi kata kunci analisis
    "analysis_keywords_absent": [
        "fundamental", "revenue", "earnings",
        "laporan keuangan", "valuasi", "PE ratio",
        "debt", "cash flow", "manajemen"
    ]
}
```

#### Sinyal Harga (Layer 2)
```python
def analyze_price_for_fomo(ticker: str) -> dict:
    
    data = yf.download(ticker, period="30d")
    
    # Hitung kenaikan dalam berbagai window
    change_5d  = (data['Close'][-1] / data['Close'][-6] - 1)
    change_10d = (data['Close'][-1] / data['Close'][-11] - 1)
    change_30d = (data['Close'][-1] / data['Close'][-31] - 1)
    
    # Volume anomaly — kenaikan volume = hype
    avg_volume = data['Volume'][:-5].mean()
    recent_volume = data['Volume'][-5:].mean()
    volume_ratio = recent_volume / avg_volume
    
    return {
        "change_5d": change_5d,
        "change_10d": change_10d,
        "change_30d": change_30d,
        "volume_ratio": volume_ratio,
        "fomo_price_signal": change_5d > 0.15 or 
                             volume_ratio > 2.0
    }
```

#### Scoring FOMO
```python
def score_fomo(text_signals, price_signals) -> float:
    score = 0.0
    
    # Text signals (bobot 40%)
    if text_signals["urgency_count"] >= 2:
        score += 0.20
    if text_signals["social_proof_count"] >= 1:
        score += 0.15
    if text_signals["analysis_absent"]:
        score += 0.05
    
    # Price signals (bobot 60%)
    if price_signals["change_5d"] > 0.20:
        score += 0.30
    elif price_signals["change_5d"] > 0.15:
        score += 0.20
    if price_signals["volume_ratio"] > 2.0:
        score += 0.20
    elif price_signals["volume_ratio"] > 1.5:
        score += 0.10
    
    return min(score, 1.0)  # cap di 1.0
```

### BIAS 2: Loss Aversion Detection

#### Definisi Operasional
Loss aversion terdeteksi ketika sinyal menunjukkan
user sedang dalam posisi rugi tetapi menolak
untuk mengakuinya namun mencari justifikasi
untuk tetap hold atau averaging down.

#### Sinyal Text (Layer 1)
```python
LOSS_AVERSION_TEXT_SIGNALS = {

    ],
    
    # Kata kunci averaging down
    "averaging_keywords": [
        "average down", "tambah lagi",
        "beli lagi biar rata", "harga murah nih",
        "kesempatan beli lebih banyak",
        "dollar cost averaging"

    # Kata kunci blame eksternal
    "blame_keywords": [
        "salah bandar", "digoreng", "manipulasi",
        "harusnya naik", "tidak wajar turun",
```

#### Sinyal Harga (Layer 2)
```python
def analyze_price_for_loss_aversion(ticker: str,
                                     buy_price: float = None
                                     ) -> dict:
    data = yf.download(ticker, period="90d")
    current_price = data['Close'][-1]
    
    # Jika buy price tidak diketahui,
    # gunakan harga 30-60 hari lalu sebagai proxy
    if not buy_price:
        proxy_buy = data['Close'][-45]  # 45 hari lalu
        unrealized_loss = (current_price / proxy_buy - 1)
    else:
        unrealized_loss = (current_price / buy_price - 1)
    
    # Cek apakah aset dalam downtrend
    ma_20 = data['Close'][-20:].mean()
    ma_50 = data['Close'][-50:].mean()
    downtrend = ma_20 < ma_50
    
    return {
        "unrealized_loss": unrealized_loss,
        "downtrend": downtrend,
        "loss_aversion_price_signal": 
            unrealized_loss < -0.10 and downtrend
    }
```

#### Scoring Loss Aversion
```python
def score_loss_aversion(text_signals, 
                         price_signals) -> float:
    score = 0.0
    
    # Text signals (bobot 50%)
    if text_signals["denial_count"] >= 2:
        score += 0.25
    if text_signals["averaging_count"] >= 1:
        score += 0.15
    if text_signals["blame_count"] >= 1:
        score += 0.10
    
    # Price signals (bobot 50%)
    if price_signals["unrealized_loss"] < -0.20:
        score += 0.30
    elif price_signals["unrealized_loss"] < -0.10:
        score += 0.15
    if price_signals["downtrend"]:
        score += 0.20
    
    return min(score, 1.0)
```

### BIAS 3: Confirmation Bias Detection

#### Definisi Operasional
Confirmation bias terdeteksi ketika pola query user
menunjukkan pencarian informasi yang searah
semua mengarah ke satu kesimpulan
tanpa ada pencarian counter-argument.

#### Sinyal Text (Layer 1)
```python
CONFIRMATION_BIAS_TEXT_SIGNALS = {

        "kenapa naik", "alasan naik",
        "prospek bagus", "potensi besar",
        "kenapa harus beli", "alasan beli"
    ],
    
    # Absensi kata kunci risiko
    "risk_keywords_absent": [
        "risiko", "downside", "bahaya",
        "kenapa turun", "alasan jual",
        "red flag", "masalah", "hutang"
    ],
    
    # Kata kunci echo chamber
    "echo_keywords": [
        "kata komunitas", "kata grup",
        "semua bilang", "rata-rata bilang",
        "konsensus", "semua setuju"
    ]
}
```

#### Scoring Confirmation Bias
```python
def score_confirmation_bias(text_signals) -> float:
    # Confirmation bias tidak butuh price signal
    # Deteksinya murni dari pola text
    
    score = 0.0
    
    if text_signals["leading_count"] >= 2:
        score += 0.35
    if text_signals["risk_absent"]:
        score += 0.35
    if text_signals["echo_count"] >= 1:
        score += 0.30
    
    return min(score, 1.0)
```

## Klasifikasi Final
```python
def classify_bias(fomo_score: float,
                  loss_aversion_score: float,
                  confirmation_score: float) -> dict:
    
    scores = {
        "FOMO": fomo_score,
        "LOSS_AVERSION": loss_aversion_score,
        "CONFIRMATION_BIAS": confirmation_score
    }
    
    # Ambil bias dengan score tertinggi
    primary = max(scores, key=scores.get)
    
    # Threshold minimum untuk trigger intervensi
    INTERVENTION_THRESHOLD = 0.40
    
    if scores[primary] < INTERVENTION_THRESHOLD:
        return {
            "bias_detected": None,
            "message": "Tidak ada pola bias signifikan 
                        yang terdeteksi."
        }
    
    return {
        "bias_detected": primary,
        "confidence": scores[primary],
        "all_scores": scores
    }
```

```python
result = {
```

## Keterbatasan Deteksi

**Yang sistem ini tidak bisa deteksi:**
- Genuine conviction yang terlihat seperti FOMO
- Context di luar teks yang ditulis user
- Bias yang tidak memunculkan sinyal text

**Cara mitigasi:**
- Threshold intervensi yang tidak terlalu rendah (0.40)
- Output selalu framed sebagai "pola yang terdeteksi"
  bukan "kamu pasti sedang X"
- User selalu bisa lanjut — sistem tidak memblokir

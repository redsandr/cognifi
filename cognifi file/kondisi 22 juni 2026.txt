ANALISIS SITUASI — Apa yang berantakan dan kenapa
    
    1. Folder structure flat (semua di root)
    
    behavioral-finance-ai/
    ├── app.py (878 baris, 44KB)
    ├── bias_detector.py (2330 baris, 109KB)
    ├── keywords.py (1332 baris, 62KB)
    ├── llm.py (409 baris)
    ├── counter_evidence.py (467 baris)
    ├── intervention.py (311 baris)
    ├── fundamental_data.py (298 baris)
    ├── news_context.py (260 baris)
    ├── price_fetcher.py (214 baris)
    ├── rag.py (627 baris)
    ├── ticker_extractor.py (376 baris)
    ├── gemini_baseline.py, config.py, evaluate_final.py...
    
    Semua file dalam satu folder. Tidak ada struktur src/, tidak ada pemisahan concerns.
    
    2. bias_detector.py = 2330 baris, 109KB
    Kode ini sudah jadi switch-case monster. Dari Layer 1 sampai Layer 4, terus ada adjustment rule A, B, C... sampe Z, AA, DD, EE, FF, GG, HH, II, JJ, KK, LL, MM, NN, OO, PP, QQ, RR, SS, TT, UU, VV, WW, XX, YY, ZZ — lebih dari 50 adjustment rules manual. Ini tidak maintainable.
    
    3. keywords.py punya duplikasi
    Beberapa keyword muncul di 2+ category sekaligus (mis. "haka aja", "siap-siap jam", "bapak prajogo" muncul di FOMO_URGENCY dan lagi di bagian bawah).
    
    4. Kontradiksi bias_detector.py vs dokumentasi cognifi:
    Dokumentasi (Bias Detection Logic): 4-layer sederhana (text → price → score → classify)
    Kode Sebenarnya: 6-layer + 50+ adjustment rules manual
    ────────────────────────────────────────
    Dokumentasi (Bias Detection Logic): INTERVENTION_THRESHOLD = 0.40
    Kode Sebenarnya: Threshold ada di code tapi banyak rule override yang langsung
      nge-boost/drop 0.5+
    ────────────────────────────────────────
    Dokumentasi (Bias Detection Logic): Scoring: text 40% / price 60% (FOMO)
    Kode Sebenarnya: Scoring sekarang jauh lebih kompleks, banyak boost 0.45-0.70 tanpa
      proporsional
    ────────────────────────────────────────
    Dokumentasi (Bias Detection Logic): Confirmation bias: detection murni dari text
    Kode Sebenarnya: Sekarang ada contextual adjustment yang campur aduk
    
    5. app.py campur aduk
    - UI Streamlit styling CSS (150+ baris cuma buat style)
    - Business logic (intent router, validation, API calls)
    - Data processing (fundamental analysis, news fetching)
    - Session management
    - Semua dalam 1 file
    
    6. llm.py vs bias_detector.py double-classification
    app.py memanggil keduanya: classify_bias_from_input() (LLM) DAN detect_bias() (rule-based). Hasilnya di-merge dengan logic max(). Ini redundan dan tidak jelas mana yang authoritative.

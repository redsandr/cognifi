Menjelaskan bagaimana sistem menyampaikan

## Prinsip Intervensi

### Prinsip 1: Friction, Bukan Blokir
Sistem tidak pernah mencegah user untuk bertransaksi.
Sistem hanya memperlambat, cukup untuk
System 2 aktif, tidak lebih.

Riset Kahneman mengkonfirmasi:
System 2 butuh waktu dan ruang untuk aktif.
Friction yang tepat menciptakan ruang itu.

### Prinsip 2: Data, Bukan Ceramah
Sistem tidak menggurui.
Sistem tidak bilang "kamu salah" atau "jangan beli."
Sistem hanya menyajikan data dan bertanya satu pertanyaan.

### Prinsip 3: Satu Pertanyaan Reflektif
Output selalu diakhiri satu pertanyaan
bukan untuk dijawab ke sistem,
tapi untuk dijawab user ke dirinya sendiri.

### Prinsip 4: Transparan tentang Sumber
Setiap angka yang muncul di output
harus bisa di-trace ke sumbernya.
Tidak ada angka yang tidak bisa diverifikasi.

## Friction Mechanism

### Apa itu Friction by Design
Setelah output intervensi ditampilkan,
user tidak langsung bisa melanjutkan.

Ada satu langkah tambahan yang harus dilakukan:
mengklik tombol konfirmasi yang menyatakan
mereka sudah membaca dan memahami data.

### Implementasi
```python
# Di Streamlit:

if intervention_shown:
    st.warning("⚠️ Bias pattern terdeteksi. 
                Baca data di atas sebelum melanjutkan.")
    
    confirmed = st.checkbox(
        "Saya sudah membaca data historis di atas "
        "dan memahami risikonya."
    )
    
    if confirmed:
        st.success(
            "Keputusan ada di tanganmu. "
            "Pastikan thesis investasimu solid."
        )
```

### Kenapa Checkbox, Bukan Timer
Timer terasa seperti hukuman, user akan frustrasi.
Checkbox adalah **active acknowledgment**
user harus secara sadar memilih untuk lanjut.

Ini menciptakan micro-commitment:
dengan mencentang, user mengakui sudah membaca data.
Secara psikologis ini lebih efektif dari timer.

## Tone Panduan

**Yang harus ada:**
- Faktual dan netral
- Berbasis data yang bisa diverifikasi
- Satu pertanyaan reflektif per output
- Disclaimer yang jelas

**Yang tidak boleh ada:**
- Kata "jangan", "salah", "berbahaya"
- Prediksi ("harga akan turun")
- Opini tentang aset ("saham ini jelek")
- Lebih dari satu pertanyaan reflektif

**Contoh tone yang salah:**
> "Kamu sedang FOMO dan ini berbahaya.
>  Jangan beli sekarang karena harga pasti koreksi."

**Contoh tone yang benar:**
> "Pola ini terdeteksi dalam inputmu.
>  Data historis menunjukkan X.
>  Apakah thesis investasimu berubah,
>  atau hanya harganya?"

## Skenario Edge Case

### Skenario 1: Tidak Ada Bias Terdeteksi
```
✅ Tidak ada pola bias signifikan yang terdeteksi
   dari input ini.

Input kamu menunjukkan:
→ Tidak ada sinyal urgensi yang kuat
→ Pertanyaan bersifat analitis
→ Kondisi harga tidak menunjukkan anomali

Tetap lakukan riset fundamental sebelum eksekusi.
```

### Skenario 2: Multiple Bias Terdeteksi
```
⚠️ Dua pola terdeteksi secara bersamaan:
   FOMO (confidence: 78%) + 
   Confirmation Bias (confidence: 65%)

Sistem menampilkan intervensi untuk bias
dengan confidence tertinggi terlebih dahulu.
```

### Skenario 3: Data Historis Tidak Cukup
```
⚠️ FOMO Pattern Terdeteksi
   Confidence: 82%

Data historis [Ticker].JK:
→ Data tersedia hanya sejak [tahun]
→ Episode serupa ditemukan: 2 kejadian
→ Sample terlalu kecil untuk probabilitas reliable

Rekomendasi: cek data fundamental secara manual
sebelum eksekusi.
```
